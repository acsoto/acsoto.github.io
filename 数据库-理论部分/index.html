绪论

数据库系统

独立性

  • 物理独立性：物理改变，逻辑不变
  • 逻辑独立性：逻辑改变，程序不变

完整性

  • 数据正确性
  • 相容性

三级模式两级映像

  • 模式：全体数据的逻辑结构和特性

  • 外模式：某一应用有关逻辑表示

  • 内模式：数据物理结构和存储方式

  • 外模式/模式映像——逻辑独立性

  • 模式/内模式影响——物理独立性

数据控制

  • 安全
  • 完整
  • 并发
  • 恢复

数据模型

要素：

  • 数据结构
  • 数据操作
  • 完整性约束

分类：

  • 层次模型
  • 网状模型
  • 关系模型

DBMS

数据库系统建立在数据库管理系统DBMS之上

功能

  • 数据库定义
  • 数据存取
  • 数据库运行管理
  • 数据组织和存储
  • 数据库建立和维护

关系数据库

概念

  • 码
    • 候选码
    • 主码
  • 主属性
  • 非主属性
  • 关系模式
  • 语义约束
    • 实体完整性
    • 参照完整性

关系数据库理论

函数依赖

Armstrong 公理

  • 自反
  • 增广
  • 传递

函数依赖闭包

$X_F^+$ X能推出的所有集合

范式

1NF

每格只有一个值

2NF

每个非主属性完全依赖于码

3NF

每个非主属性不传递依赖于码

BCNF

对于R的每个函数 依赖X->Y，且Y$\nsubseteq$X时，X必含有码

  • 非主属性完全依赖于每个候选码
  • 主属性完全依赖于不包含它的候选码
  • 没有任何属性完全依赖于非码

规范化

求候选码

分为L，R，N，LR四类，L和N为主属性，不断加入LR中的一个属性，到能推出全集

判断范式

看完非主属性是否完全依赖和传递依赖

3NF的等价分解

保持函数依赖

  1. 极小化FD
    1. 拆右侧，去掉重复的
    2. 拆左侧，看是否还能推出右侧，决定是否取代
    3. 遍历一遍，去掉多余的
  2. 不在F中出现的属性构成一个关系模式
  3. 如果有X->A，XA直接构成全集，不能分解，算法终止
  4. 对F按照左边相同分组，分别把左右构成集合，删掉被包涵的

查询优化

分为代数优化和物理优化

全局查询是分布式数据库查询优化的主要对象

连接查询优化技术：半连接

分布式数据库系统中，通过全局模式和分片模式之间的映像实现分片透明性

事务处理技术

事务

用户定义的操作序列，是数据库恢复和并发的基本单位

特性：ACID

  • 原子性 Atomicity 操作要么全做要么不做
  • 一致性 Consistency 从一个一致性状态转移到另一个
  • 隔离型 Isolation 并发执行互不干扰
  • 持久性 Durability 一旦提交数据修改就是永久性的

事务故障恢复 UNDO

  1. 反向扫描日志
  2. 进行更新操作的逆操作

系统故障 UNDO+REDO

  1. 正向扫描日志，找到故障前已提交事务，加入REDO队列，未完成事务，加入UNDO队列
  2. UNDO
  3. REDO

并发控制

合理调度事务，避免不一致

封锁

  • 排它锁 X
  • 共享锁 S

封锁协议

  • 一级：修改前加X锁，事务结束释放，防止丢失修改
  • 二级：一级基础上读前加S锁，读完释放，防止读脏
  • 三级：二级基础上，事务结束才释放，保证数据不可重复读

封锁粒度

多粒度封锁协议：对单个节点加锁则其后裔节点也加同类型的锁

意向锁

该节点下层节点正在被加锁，对任意节点解锁时，必须对上级节点加意向锁。

  • 意向共享锁（IS锁）
  • 意向排它锁（IX锁）
  • 意向共享排它锁（SIX锁）：加S锁，再加IX锁。

死锁

预防死锁

  • 一次封锁法
  • 顺序封锁法

检测死锁

  • 超时法
  • 等待图法

可串行性

并发执行与按某一次串行执行结果相同

两端锁协议可以保证事务并发操作的正确性

数据库保护

存取控制方法

  • 强制存取控制(MAC)
  • 自主存取控制(DAC)

为维护完整性，DBMS应提供定义完整性约束条件的机制，提供完整性检查方法，具有违约响应

分布式数据库

分布式数据库以数据分布为前提，强调场地自治性和协作性

分布透明性包括分片透明性，位置透明性，局部数据模型透明性
