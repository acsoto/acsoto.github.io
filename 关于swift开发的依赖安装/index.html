关于Swift开发的依赖安装

Dec 27, 2021

今天IOT课写报告突然要用到[[IoTMonitor]]，发现打开Xcode无法编译，找到原因是依赖SwiftHTTP出现了问题，缺少了其中的文件，鉴于之前直接下载了源文件拖进我的工程里，确实不是个规范操作方法，同时，之前做这个app遇到了各种新奇的依赖安装方式，于是决定重新规范安装一下，顺带记录。

CocoaPods

这个应该是比较常用的方式？首先安装CocoaPods。在项目目录下创建Podfile文件，写入依赖所需的内容，例如


source 'https://github.com/CocoaPods/Specs.git'
platform :ios, '8.0'
use_frameworks!

pod 'SwiftHTTP', '~> 3.0.1'

后 pod install

虽然但是，我隐约记得我上次用这个方法搞不定。

Carthage

这次我用了这个，首先用brew安装：brew install carthage

在项目目录下创建Cartfile文件，写入依赖所需内容，如

github "daltoniam/SwiftHTTP" >= 3.0.1

Run carthage update --use-xcframeworks

进入生成的Carthage目录Carthage/Build，把.xcframework直接拖入项目的Frameworks区域即可。

Xcode

另外就是Xcode自带的装依赖方法了，一般如果依赖支持都会写操作步骤

用ChartView举例

File → Add Packages…

搜索项目地址 https://github.com/AppPear/ChartView

顺便推荐一下这个ChartView，真的很好看。
